# SEO page generator — CalorIA Scan

Generates bilingual (EN + ES) "calories in X" pages from the dishes database,
plus `sitemap.xml` and `robots.txt`. Built for Mexican & Latino food — the
keywords where a DR-0 site can actually rank (validated in Google Keyword Planner).

## Files
- `mappings.json` — 61 head concepts → DB match pattern + target keyword (EN/ES). The curated backlog.
- `pages.json` — data snapshot (nutrition baked in). Generated by `fetch-dishes.js`.
- `fetch-dishes.js` — refresh `pages.json` from Supabase (zero deps, Node 18+).
- `generate.js` — render `pages.json` → HTML + sitemap + robots.

## Refresh data from Supabase (optional)
```bash
SUPABASE_URL=https://<ref>.supabase.co SUPABASE_KEY=<anon-or-service-key> node seo/fetch-dishes.js
```
Use the **anon** key if `dishes` allows public read (preferred). Never commit a
service_role key — keep it in a `.gitignored` `.env`.

## Generate pages (writes into the repo root)
```bash
node seo/generate.js .          # outputs en/calories/, calorias/, sitemap.xml, robots.txt
```
(Default output dir is `./build` for previewing; pass `.` to write into the site root.)

## After deploy
1. Submit `https://caloriascan.com/sitemap.xml` in Google Search Console.
2. Add a footer/nav link to `/en/calories/` (EN) and `/calorias/` (ES) so crawlers find the hub.
3. Expand: add rows to `mappings.json` (LATAM dishes — Argentina, Colombia, Brasil…) and re-run.
